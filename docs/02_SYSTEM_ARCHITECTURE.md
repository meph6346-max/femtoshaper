# 02. 시스템 아키텍처

## 전체 블록 다이어그램
```
┌─────────────────────────────────────────────────────┐
│  브라우저 (모바일/PC)                                 │
│  ┌─────────────────────────────────────────────┐    │
│  │  index.html — 4탭 SPA                       │    │
│  │  15개 JS 모듈 + Chart.js + CSS              │    │
│  │                                              │    │
│  │  쉐이퍼탭 │ 진단탭 │ 라이브탭 │ 설정탭      │    │
│  └─────────────────┬───────────────────────────┘    │
└────────────────────│────────────────────────────────┘
                     │  HTTP/JSON + SSE (WiFi)
┌────────────────────│────────────────────────────────┐
│  ESP32-C3          │                                 │
│  ┌─────────────────▼───────────────────────────────┐│
│  │ WebServer (port 80)                              ││
│  │  REST API (25 endpoints) + SSE + LittleFS        ││
│  └──────────────────┬──────────────────────────────┘│
│  ┌──────────────────▼──────────────────────────────┐│
│  │ 상태머신 (IDLE → PRINT → DONE)                   ││
│  └──────────────────┬──────────────────────────────┘│
│  ┌──────────────────▼──────────────────────────────┐│
│  │ DSP 엔진 (dsp.h)                                ││
│  │ FFT 1024pt → PSD → 듀얼 X/Y 투영 → 에너지 게이트││
│  └──────────────────┬──────────────────────────────┘│
│  ┌──────────────────▼──────────────────────────────┐│
│  │ ADXL345 드라이버 (SPI 5MHz, FIFO 32샘플 폴링)    ││
│  └─────────────────────────────────────────────────┘│
│  ┌─────────────────────────────────────────────────┐│
│  │ NVS 저장소 (6 네임스페이스, ~2KB/32KB)            ││
│  └─────────────────────────────────────────────────┘│
└──────────────────────────────────────────────────────┘
```

## 계산 분담
| 역할 | 위치 | 이유 |
|------|------|------|
| 가속도 수집 | ESP32 | SPI 하드웨어 |
| FFT + PSD | ESP32 (dsp.h) | 실시간 처리 필수 |
| 에너지 게이트 | ESP32 (dsp.h) | 세그먼트별 즉시 판단 |
| 피크 검출 | JS (filter.js) | 8피크 분류 로직 복잡 |
| 쉐이퍼 계산 | JS (shaper.js) | 5종×주파수 스윕 = RAM 부족 |
| 진단 | JS (diagnostic.js) | 키네마틱별 규칙 테이블 |
| 판정 | JS (validator.js) | 2계층 품질평가 |
| UI 렌더링 | JS (charts.js + app.js) | 브라우저 Chart.js |

## 통신 프로토콜
- **REST API:** HTTP GET/POST, Content-Type: application/json
- **SSE:** 라이브 스펙트럼, `text/event-stream`, 단방향
- **CORS:** 없음 (같은 ESP32에서 서빙)
- **동시 접속:** 1 클라이언트 (SSE + WebServer)

## 파일 서빙
ESP32 LittleFS에 프론트엔드 파일 저장. WebServer가 정적 파일 서빙:
```
/            → /index.html
/style.css   → data/style.css
/app.js      → data/app.js
/chart.min.js → data/chart.min.js (201KB)
...
```
